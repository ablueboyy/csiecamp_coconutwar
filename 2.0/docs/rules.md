# 🥥 COCONUT WARS 2.0 · 開墾版 完整規則書

> 一款島嶼佔領 × 資源養成的策略對戰。目標很單純：**回合結束時，椰子（分數）最多的小隊獲勝。**
> 本版本為「第一天剛認識的高中生」設計，移除了需要跨隊精確協調的協防與結盟，指令只有 **開墾 / 移動 / 攻擊** 三種。

---

## 一、名詞解釋

| 名詞 | 說明 |
| :--- | :--- |
| **小隊** | 玩家隊伍，編號 **0 ~（小隊數−1）**（例如 10 隊 = 第 0 隊 ~ 第 9 隊）。 |
| **兵力** | 小隊的行動資源，如同「錢」。用來開墾、移動、進攻。以 **100 為最小單位**。 |
| **椰子** | 分數。**只會累加、不會被扣**，最後比誰多。 |
| **島嶼** | 地圖上的據點，分**大島**（5 座）與**小島**（7 座），共 12 座。持有島嶼每回合會產出椰子。 |
| **專屬兵營** | 每隊在地圖下方沙灘的家。兵營裡的兵是「未部署」的機動兵力，**絕對安全、不可被攻擊**。 |
| **駐軍** | 留在某座島上的兵力，代表該島的防守力量，也是資源分配與被攻擊時的守方。 |

---

## 二、賽前設定

1. **輸入小隊數（預設 10）與回合數（預設 4）**。
2. **島嶼歸屬**：設定每座島一開始由哪一隊佔領（沿用前置大地遊戲的結果），或設為中立。
3. **各隊累積總兵力**：輸入每隊在大地遊戲累積的總兵力。
4. **初始兵力分發**（系統自動計算）：
   - 擁有**大島**者，該島上自動放 **1000** 駐軍；擁有**小島**者放 **500** 駐軍。
   - 每隊「總兵力 − 島上駐軍」的剩餘，全部放入**專屬兵營**。

> 地圖上每座島會**持續顯示**目前佔領的小隊與兵力；地圖下方沙灘顯示每隊兵營兵力。

---

## 三、回合流程

- 遊戲共進行設定的回合數（預設 4 回合）。
- **每個小隊每回合最多提交 3 個指令**（開墾 / 移動 / 攻擊，可重複類型）。
- 所有小隊指令提交後，主控台按「結算本回合」，系統**依固定順序**處理並播放動畫：

> **結算順序：① 開墾 → ② 移動 → ③ 攻擊 → ④ 資源發放**

三個行動階段**依序執行**：先處理完所有開墾，再處理所有移動，最後處理所有攻擊。因此**同一回合內可以「先移動、再從新位置發動攻擊」**（因為移動先於攻擊結算）。

---

## 四、指令總覽

所有出發地（S）必須是**己方的兵營或己方島嶼**。兵力一律為 **100 的倍數**。

### 1. 🌱 開墾 `開墾(出發, 己方島, 兵力)`
- 把兵力投入**自己的**一座島，這些兵**永久消耗**（不再是可用兵力）。
- 效果：該島**每回合椰子產出永久增加**——**每投入 100 兵 → +30 椰子／回合**。
  - 例：投入 300 兵 → 該島每回合多產 +90 椰子。
- **加成綁在島上**：若這座島之後被別人攻下，開墾加成會**隨島留給新主人**（所以高產島要顧好，否則等於幫對手養）。

### 2. 🚶 移動 `移動(出發, 目的, 兵力)`
- 在**自己的兵營與自己的島嶼之間**調度兵力。
- 目的地必須是己方兵營或己方島嶼。

### 3. ⚔️ 攻擊 `攻擊(出發, 目標, 兵力)`
- 對**不是自己的**島嶼發動進攻。
- **最低門檻 300 兵**，且為 100 的倍數。
- **同一隊可以從多個出發地、對同一座島發動多次攻擊**；系統會**先把該隊投入同一島的兵力全部加總**，再進行結算。

> 提交介面提供「🖱️ 用選的（下拉選單）」與「⌨️ 用打的（打字，如 `兵營, 大島2, 300`）」兩種方式，任選其一。

---

## 五、開墾階段結算（①）

- 依序處理所有開墾指令：從出發地扣除投入兵力（永久消耗），該島的每回合椰子加成增加 `floor(投入 ÷ 100) × 30`。
- 開墾的是**自己**的島；若目標非己方領地，該指令作廢。

---

## 六、移動階段結算（②）

- 依序處理所有移動指令：從出發地扣除、加到目的地。
- 目的地必須是己方兵營或己方島嶼，否則作廢。

---

## 七、攻擊階段結算（③）— 核心裁決

先把每一隊「對同一座島」的所有進攻兵力**加總**成一支部隊，再對**每一座被攻擊的島**進行以下裁決。

參與者：**各進攻方**（每隊一支，已加總）＋ **守方**（該島目前的駐軍）。

### 第一階段：攻方內鬥，選出「突圍者」
反覆進行：
1. 找出目前**兵力最高**的進攻方。
2. **若有多支進攻方同為最高（平手）** → 這些平手部隊**互相抵銷**：
   - **一半兵力直接消失、另一半撤回各自兵營**，並**喪失本回合攻擊資格**（退出這場爭奪）。
   - 例：兩支 2000 平手 → 各損 1000（消失）、各留 1000（撤回兵營）。
3. 移除平手部隊後，往下一層繼續比。
4. 直到出現**唯一最高的進攻方** → 它就是**突圍者（成功殺到守軍面前的那一隊）**。
   - 此時仍留在場上、但兵力較低的其他進攻方 → **未突圍，進攻失敗，全數陣亡**。
5. 若所有進攻方都在互相抵銷中出局 → **沒有突圍者**（見下方）。

### 第二階段：突圍者 vs 守軍
- **突圍者 > 守軍**：突圍者**攻下該島**，其兵力**扣掉守軍兵力**後留駐；**守軍全部陣亡**。
- **突圍者 < 守軍**：守方**守住**，其兵力**扣掉突圍者兵力**後留駐；**突圍者全部陣亡**。
- **突圍者 = 守軍**：**同歸於盡**，兩邊全滅，該島變**無人空島**。
- **該島原本沒有守軍（0 兵）**：突圍者**直接佔領、不扣兵**。
- **沒有突圍者（攻方全數抵銷）**：守方**原封守住**；若原本就沒有守軍，該島成無人空島。

### 兵力去向總整理
- **抵銷出局**的進攻方：一半消失、**一半撤回兵營**（下回合可再用）。
- **未突圍的較弱進攻方**：**全滅**（投入的兵全部損失）。
- **戰敗的守方**：**駐軍全部消失**。
- **勝方**：扣掉對手兵力後的剩餘，留在該島當新駐軍。

### 完整範例
A、B、C、D、E 五隊都攻擊 F 的島；F 島上守軍 300：

| 小隊 | 投入兵力 |
| :---: | :---: |
| A | 2000 |
| B | 2000 |
| C | 500 |
| D | 500 |
| E | 400 |
| F（守） | 300 |

裁決過程：
1. 最高為 **2000（A、B 平手）** → A、B 互相抵銷，**各損 1000、各撤回 1000**，出局。
2. 剩下最高為 **500（C、D 平手）** → C、D 互相抵銷，**各損 250、各撤回 250**，出局。
3. 剩下 **E（400）為唯一最高 → 突圍者**。
4. E(400) vs 守軍 F(300)：400 > 300 → **E 攻下該島**，扣掉守軍 300 後**留 100 兵駐守**；**F 的 300 守軍全滅**。

> 結果：該島換 **E 佔領、島上 100 兵**；A、B 兵營各回補 1000，C、D 兵營各回補 250；F 守軍歸零。

---

## 八、資源發放（④）與計分

攻擊結算完成後，各島依**最終佔領者**發放椰子：

- **大島**：每回合基礎 **500** 椰子。
- **小島**：每回合基礎 **300** 椰子。
- **加上該島的開墾加成**（每回合 +30／每 100 投入兵）。
- **豐收加成**（見下）。
- 全部椰子加入**該島佔領者**的分數（無人空島不產出）。

> 每座島只有單一佔領者（本版本沒有共同持有），所以椰子全歸島主。

---

## 九、豐收機制 🌾

- **每回合開始時，隨機指定 2 座島為「豐收島」**：該回合這 2 座島的椰子產出 **×1.5**（含開墾加成一起放大）。
- **最後一回合：全島大豐收**，所有島產出 **×1.5**。
- 豐收島會在地圖上以金光與 `🌾×1.5` 標記，並在畫面頂端公告，讓各隊在下指令前就能針對豐收島搶佔或加碼。

> 設計用意：豐收讓每回合的價值焦點浮動、鼓勵爭奪；最後一回合全體加倍，領先者無法提早躺贏，留下翻盤空間（可搭配前期養兵、最後一回合梭哈搶島的策略）。

---

## 十、勝負判定

- 所有回合結束後，**椰子總數最多的小隊獲勝**，依椰子由多到少排名。

---

## 十一、其他判定與防呆

- **兵力超支**：同一隊在同一階段、從同一個出發地移出的兵力總和，若超過該地當前兵力，涉及該出發地的這些指令**全部作廢**（兵力留在原地）。
- **攻擊自己的島**：無效，作廢。
- **開墾／移動到非己方領地**：無效，作廢。
- **兵營絕對安全**：兵營不能被攻擊，也不能被開墾／移動搶走。

---

## 十二、操作介面（雙視窗）

- **主控台**（工作人員）：設定遊戲、逐隊輸入指令、按鈕結算。
- **播放視窗**（給小隊員看）：右上「🖥️ 開啟播放視窗」另開，只顯示大地圖、兵營、椰子排行與結算動畫，**沒有任何操作鈕**。
- 兩視窗需在**同一台電腦、同一瀏覽器**內開啟，透過瀏覽器頻道即時同步；主控台一結算，播放視窗會同步播放**分階段動畫**與左下角的**「📢 結算實況」**逐條戰報。
